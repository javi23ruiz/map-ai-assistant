---
name: git-commit
description: Create git commits. Use when committing changes.
---

When creating commits:
1. Run `git status` and `git diff HEAD` to understand what changed
2. Write a commit message following Conventional Commits format:
   - feat: a new feature
   - fix: a bug fix
   - chore: maintenance tasks
   - docs: documentation changes
3. Keep the subject line under 72 characters
4. Stage only related files together — don't bundle unrelated changes.
